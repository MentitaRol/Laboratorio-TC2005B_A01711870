¿Cuál es el propósito de los métodos HTTP: GET, HEAD, POST, PUT, PATCH, DELETE?

  1. GET: Solicita al servidor una inofrmación concreta. Las peticiones de GET solo deben recuperar datos
  2. HEAD: Al igual que el GET solicita información al servidor, pero sin el cuerpo de la solicitud
  3. POST: Es usado para enviar información al servidor, esta información se guarda en el cuerpo de la solicitud
  4. PUT: Actualiza o reemplaza los recursos en el servidor
  5. PATCH: Aplica modificaciones parciales a un recurso del servidor
  6. DELETE: Borra un recurso específico del servidor

Métodos de petición HTTP - HTTP | MDN. (2023, 24 julio). MDN Web Docs. https://developer.mozilla.org/es/docs/Web/HTTP/Methods

¿Qué método HTTP se debe utilizar al enviar un formulario HTML, por ejemplo cuando ingresas tu usuario y contraseña en algún sitio? ¿Por qué?

Se debe de utilizar el metodo POST ya que se esta enviando una solicitud al servidor y por la naturaleza del metodo POST este no muestra datos sensibles en la URL lo que previeene filtración de datos delicados

El método GET ya que es el encargado de solicitar y obtener la información del servidor

Nos indica que el servidor ha procesado correctamente la solicitud del cliente y está devolviendo el contenido solicitado en el cuerpo de la respuesta.

¿Es responsabilidad del desarrollador corregir un sitio web si un usuario reporta que intentó acceder al sitio y se encontró con un error 404? ¿Por qué?

Depende del caso en el que este sucediendo este eror. Segun Google "solucionar los errores 404 no merece la pena porque este tipo de error no perjudica ni a la indexación ni al posicionamiento de tu sitio." Pero depende mucho del porque sucede este error, si bien en muchos casos sucede porque se elimino el contenido de la pagina web o porque se traslado a otro sitio, tambien existe la posibilidad de que este error surga por problemas en la URL o porque hay un problema con el servidor, en estos casos es importante solucionar los errores.


Errores 404 (Page Not Found) de página no encontrada - Ayuda de Search Console. (s. f.). https://support.google.com/webmasters/answer/2445990?hl=es

¿Es responsabilidad del desarrollador corregir un sitio web si un usuario reporta que intentó acceder al sitio y se encontró con un error 500? ¿Por qué?

El error 500 significa que hay un error interno en el servidor, por lo que el problema no viene desde el usuario. Si es responsabilidad del desarrollador corregir el sitio para solucionarel error ya que no solo causa una mala experiencia para el usuario si no que tambien puede afectar cualquier funcionalidad del sitio web

¿Qué componentes de estructura y estilo tiene una tabla?

Componentes:

  • table: Contenedor de la tabla
  • tr: Filas
  • th: Columnas
  • td: Celda de datos
  • thead: Encabezados
  • tbody: Datos principales
  • tfoot: Footer de la tabla

Estilos:

  • border: bordes de la tabla
  • border-collapse: Combina las celdas de la tabla
  • padding y margin: Espaciado interno y externo
  • width y height: Tamaño de la tabla y las celdas
